User Guide

Everything you need to build world-class web applications using browser.js — the framework already running in every browser on the planet.

What is browser.js?

browser.js is a zero-byte JavaScript framework that exposes the complete Web Platform API. It is pre-installed in every web browser and requires no download, no build step, and no configuration.

The companion stylesheet, browser.css, provides the full CSS cascade, layout engine, and every CSS property standardized to date — also at 0 bytes.

Good to know: If you have ever opened a browser and written JavaScript, you have already been using browser.js. This guide simply formalizes that relationship.

Philosophy

Modern browsers implement a vast, standardized, and heavily optimized API surface. The WHATWG, W3C, TC39, and IETF maintain these standards collaboratively, with input from every major browser vendor. The result is a platform that:

  • Receives continuous improvements with no action required from developers
  • Is optimized at the engine level — no JavaScript wrapper can be faster
  • Ships security patches globally without requiring users to update your app
  • Carries no supply-chain risk from third-party dependencies

browser.js formalizes the decision to use this platform directly, deliberately, and with zero overhead.

Installation

Option 1 — CDN

For projects that require an explicit dependency declaration, include browser.js via the jsDelivr CDN: 

<!-- JavaScript (0 bytes, loads instantly) -->
<script src="https://cdn.jsdelivr.net/gh/peteralieber/browser@main/browser.js"></script>

<!-- CSS (0 bytes, loads instantly) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/peteralieber/browser@main/browser.css">
The CDN files are exactly 0 bytes. Network transfer time is effectively 0 ms, subject to the irreducible lower bound of the speed of light and your ISP.

Option 2 — npm

npm install browser
# or: yarn add browser | pnpm add browser | bun add browser

After installation, your node_modules directory will contain the browser.js package. The size of the framework portion is 0 bytes.

Option 3 — Pre-installed (Recommended)

browser.js is pre-installed in your browser. No further steps are required. 

// Verify installation
typeof window       // → "object"  ✓
typeof document     // → "object"  ✓
typeof fetch        // → "function" ✓
// browser.js confirmed.

Quick Start

The following is a complete browser.js application: 

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My App</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <script>
    // browser.js is already available.
    document.querySelector('h1').textContent = 'Hello, browser.js!';
  </script>
</body>
</html>

DOM Manipulation

browser.js provides the complete DOM API. The following examples demonstrate common operations:

Selecting Elements

// Single element
const el = document.querySelector('#my-id');

// Multiple elements
const items = document.querySelectorAll('.list-item');

// Live HTMLCollection
const inputs = document.getElementsByTagName('input');

Modifying the DOM

// Create and append an element
const div = document.createElement('div');
div.className = 'card';
div.textContent = 'New card';
document.body.appendChild(div);

// Or with innerHTML / insertAdjacentHTML
document.querySelector('#app').innerHTML = `
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
`;

Events

// Add an event listener
document.querySelector('button').addEventListener('click', (e) => {
  console.log('Clicked!', e.target);
});

// Custom events
const event = new CustomEvent('my-event', { detail: { foo: 'bar' } });
document.body.dispatchEvent(event);

// Delegation pattern
document.addEventListener('click', (e) => {
  if (e.target.matches('.item')) handleItem(e.target);
});

Data Fetching

// GET request
const res = await fetch('https://api.example.com/data');
const data = await res.json();

// POST request with JSON body
await fetch('/api/items', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'New item' }),
});

// Streaming response
const stream = await fetch('/api/stream');
const reader = stream.body.getReader();

Storage

// localStorage (synchronous, survives page close)
localStorage.setItem('key', 'value');
const val = localStorage.getItem('key');

// sessionStorage (survives page reload)
sessionStorage.setItem('session', 'data');

// IndexedDB (async, structured, large data)
const db = await indexedDB.open('my-db', 1);

// Cache API (for service workers)
const cache = await caches.open('v1');
await cache.add('/index.html');

CSS & Styling

browser.css provides the complete CSS engine. Use it to access any CSS feature standardized or in-progress, without waiting for a library update. 

/* browser.css: Already loaded */

/* CSS Custom Properties */
:root { --primary: #4f46e5; }
.button { background: var(--primary); }

/* CSS Grid */
.layout { display: grid; grid-template-columns: 1fr 2fr; }

/* CSS Subgrid */
.child { grid-column: subgrid; }

/* Container Queries */
@container (min-width: 600px) { .card { flex-direction: row; } }

/* CSS Layers */
@layer base, components, utilities;
Any new CSS feature shipped by browser vendors is automatically available in browser.css the moment the user updates their browser. No library update required.

Web Workers

// main.js
const worker = new Worker('worker.js');
worker.postMessage({ task: 'compute', data: [1, 2, 3] });
worker.addEventListener('message', (e) => console.log(e.data));

// worker.js
self.addEventListener('message', (e) => {
  const result = e.data.data.reduce((a, b) => a + b, 0);
  self.postMessage(result);
});

WebGL & WebGPU

// WebGL 2
const canvas = document.querySelector('canvas');
const gl = canvas.getContext('webgl2');

// WebGPU (where supported)
const adapter = await navigator.gpu.requestAdapter();
const device  = await adapter.requestDevice();

WebSockets

const ws = new WebSocket('wss://api.example.com/ws');

ws.addEventListener('open',    () => ws.send('hello'));
ws.addEventListener('message', (e) => console.log(e.data));
ws.addEventListener('close',   () => console.log('closed'));

Web Crypto

// Generate a random AES-GCM key
const key = await crypto.subtle.generateKey(
  { name: 'AES-GCM', length: 256 },
  true,
  ['encrypt', 'decrypt']
);

// Generate random bytes
const bytes = crypto.getRandomValues(new Uint8Array(32));

WebAssembly

// Fetch and compile a .wasm module
const { instance } = await WebAssembly.instantiateStreaming(
  fetch('module.wasm')
);
const result = instance.exports.add(1, 2); // → 3

Full API Reference

browser.js exposes every API in the Web Platform. The complete, authoritative reference is maintained by the WHATWG and MDN Web Docs:

  • MDN Web Docs — Web APIs
  • WHATWG HTML Living Standard
  • WHATWG DOM Living Standard
  • TC39 ECMAScript Specification
  • W3C CSS Specifications

We defer all API documentation to these upstream sources. They are more comprehensive than anything we could write, and they are automatically kept up to date with each new browser.js release.

TypeScript users: the DOM type definitions bundled with TypeScript (lib.dom.d.ts) provide full type coverage for the browser.js API. No @types/browser package is necessary or available.

FAQ

Do I need to import browser.js?

No. It is already present in every browser. You may include the CDN tag if your project requires an explicit dependency manifest, but it has no functional effect.

What version should I use?

The version of browser.js in use is the version of your user's browser. You do not control this and you do not need to. All versions are compatible with the Web Platform standards.

How do I update browser.js?

browser.js is updated automatically when your users' browsers update. No action is required. You will never need to read a migration guide, run npm update, or modify your application to receive new features.

Is browser.js tree-shakeable?

browser.js is already 0 bytes. There is nothing to shake.

Can I use browser.js with React / Vue / Svelte / Angular?

Yes. All of those frameworks run on top of browser.js. You have always been using browser.js alongside them.

Does browser.js support Server-Side Rendering?

Node.js 18+ and Deno include a growing subset of the Web Platform API. Full SSR support is available in any environment that ships a browser-compatible runtime (e.g., via jsdom or happy-dom).

How do I report a bug in browser.js?

Bugs in browser.js are bugs in the browser engine. Please report them to Chromium, Mozilla, or WebKit as appropriate. Bugs in your own code are outside the scope of this project.

Can I contribute new features?

If you have implemented a new feature in your own JavaScript engine or browser fork, we will include it in browser.js immediately, for free, with no update required on your part or ours. Simply ship your browser and it will be in the next "release."
